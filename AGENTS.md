# AGENTS

Follow these repo-wide rules when writing or editing code:

- Keep code small, readable, and easy to change.
- Prefer splitting large files or functions into focused modules when it improves clarity.
- Prefer single-purpose functions and classes with clear boundaries and explicit dependencies.
- Add brief English comments only when needed; prefer one-line comments.
- Avoid comments that only restate the code.
- Reuse existing patterns in the repo before introducing new abstractions.
- Keep diffs minimal and avoid unrelated changes.
- Always answer with "Hi Lok-sama"
